This API contains data from the Häme Events form, and all other event organizers in Tavastia region.

The API is originally developed by the City of Helsinki Open Software Development team , who also provide the complete API specifications. This API instance is further developed and edited to suit the needs of the Tavastia region. This API is called Häme Events. Location information is fetched from Google Maps (at the time of writing) with extra information provided by user. In the API, you can search data by date or location as well as bounding box.

3. Henkilötiedot

Häme Events -tapahtumarajapintaan kerätään ja talletetaan tapahtuman ylläpitäjän sähköpostiosoite tapahtuman luonnin yhteydessä. Tätä sähköpostiosoitetta käytetään palvelun omistajan toimesta ainoastaan tapahtumaa koskevien ilmoitusten lähettämiseen tapahtuman ylläpitäjälle (Esimerkiksi virheellisestä tapahtumatiedosta tulleet ilmoitukset) tai tapahtumakohtaisen salasanan palauttamiseen. Tätä tietoa ei näytetä Häme Events -palvelussa eikä sitä myöskään saa haettua rajapinnasta muilla tavoin.

Mikäli tapahtumatietojen yhteyteen lisätään muita henkilötietoja (esimerkiksi tapahtuman yhteyshenkilön nimi ja yhteystiedot), vakuutan, että minulla on oikeus ja asianomaisten henkilöiden suostumus lisätä tapahtumatietoihin tapahtumalle tarpeellisten henkilöiden tiedot. Henkilötietoja saa käyttää ainoastaan tapahtumaa käsittelevään tiedotukseen ja viestintään. Henkilötietojen käyttö tai siirto muihin tarkoituksiin on ehdottomasti kielletty.

Rajapinnan ylläpitäjä ei käytä, luovuta tai myy käyttäjien lisäämiä henkilötietoja tai käytä niitä muihin kuin Häme Eventsin välttämättömiin ylläpidollisiin tarkoituksiin (pl. viranomaisten tiedonsaantioikeus).

Tapahtumaan liittyvät ilmoitukset

Tapahtumasta lähetettävä ilmoitus tulee lähettää api.tavastiaevents.fi/tavastiaevents/report/ osoitteeseen POST-pyynnöllä. Pyyntöön pitää liittää message_code ja event_id tiedot. Lisänä voidaan lähettää message, jossa voidaan lähettää lisätietoa liittyen pyyntöön, joka liitetään lähetettävään sähköpostiin. message tiedon maksimipituus on 300 merkkiä.

Tiedot lähetetään form-datana

Jos viestin pituus ylittää 300 merkkiä, rajapinta palauttaa 'message': "Message length can't exceed 300 characters"
Jos viestistä puuttuu message_code tai event_id, rajapinta palauttaa 'required': 'event_id and message_code must be suplied with the report'
Jos ilmoitettua tapahtumaa ei löydy, rajapinta palauttaa 'event_id': 'Event with that id was not found'
Jos lähetettiin virheellinen message_code, rajapinta palauttaa'message_code': 'Message code was not found on the server'
Jos viesti lähetettiin onnistuneesti, rajapinta palauttaa 'message_sent': 'Successful'
Jos lähetyksessä on tapahtunut virhe rajapinnan puolella, rajapinta palauttaa 'server_error': 'Error occurred on server. Please try again'

Virhe tapahtuman tiedoissa

message_code, jonka arvona 100
event_id, jonka arvona ilmoitettavan tapahtuman id
messageMahdollinen viesti lisätään lähetettävään sähköpostiin.
Tästä lähtee sähköposti ylläpitäjälle ja tapahtuman luojan ilmoitettuun sähköpostiin.
